Accessibility

Accessibility statement

Last updated: 25 May 2026

Our commitment

Aura Glossy is committed to ensuring digital accessibility for people with disabilities. We are continuously improving the user experience for everyone and applying the relevant accessibility standards.

Standards we follow

Aura Glossy works toward conformance with the Web Content Accessibility Guidelines (WCAG) 2.1 Level AA and the Israeli accessibility standard IS 5568. We use these standards as a baseline for every design and engineering decision.

Supported accessibility features

  • Keyboard navigation — all interactive elements (links, buttons, hotspot dots, modals, form fields) are reachable and operable using only a keyboard.
  • Skip-to-content link — a hidden link at the top of every page lets keyboard users jump directly to the main content.
  • Visible focus indicators — keyboard focus is always shown with a warm-gold outline that doesn't change visual layout.
  • Reduced motion — when your device requests reduced motion (macOS / iOS "Reduce motion", Windows "Show animations off"), Aura Glossy disables non-essential animations and transitions.
  • Screen reader support — semantic HTML, ARIA labels on icon-only controls, dialog and live-region announcements for modals and toasts.
  • Forced-colors / High Contrast Mode — focus indicators and key UI elements respect Windows High Contrast and macOS Increase Contrast settings.
  • Language and direction — Aura Glossy supports English, Spanish, Arabic, and Hebrew with correct lang and dir attributes (Arabic and Hebrew render right-to-left).
  • Font scaling — page layout responds to user-controlled text size in the browser and operating system.
  • Mobile and touch — touch targets meet recommended minimum sizes; hotspot dots and menu controls work on touch without hover.

Browser and assistive technology support

Aura Glossy is tested on the latest stable versions of:

  • Safari (macOS, iOS) with VoiceOver
  • Chrome (Windows, macOS, Android) with NVDA / TalkBack
  • Firefox (Windows, macOS)
  • Edge (Windows) with Narrator

Older browsers and in-app browsers (Instagram, TikTok, Facebook) are functionally supported but may have reduced visual polish. We recommend opening Aura Glossy in your default browser for the best experience.

Known limitations

We are transparent about areas where work is still in progress:

  • Some outfit images use dynamic alt text generated from product names; we are gradually adding more descriptive curator-written captions.
  • Linked third-party retailer sites (Zara, H&M, COS, Modanisa, etc.) have their own accessibility levels that we cannot control.
  • A small number of legacy animations use older patterns; they respect the system reduced-motion preference but may not fully pause in every browser.

Continuous improvement

Accessibility is an ongoing process, not a finish line. Every new feature on Aura Glossy is reviewed against WCAG 2.1 AA before it ships. We re-audit the live site on a recurring schedule.

Contact for accessibility issues

If you encounter an accessibility barrier on Aura Glossy — anywhere on the site — please tell us. We will respond within five business days and prioritise the fix.

Email: auraglossy.support@gmail.com

Please include: the page URL, what assistive technology you were using, and a short description of the problem.
